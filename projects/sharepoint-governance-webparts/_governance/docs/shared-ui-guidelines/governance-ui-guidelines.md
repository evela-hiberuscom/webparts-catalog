# Shared UI guidelines

- Use Fluent UI v8 components and SharePoint theme-friendly surfaces.
- Always show loading, empty, error, mock, backend-required and limitations states.
- Use MessageBar for mock/backend limitation warnings.
- Do not expose destructive buttons in SPFx until a secure backend supports dry-run, approval, audit and rollback planning.
- Do not rely on color alone; include severity text.
- Keep dashboards useful for non-technical administrators.
